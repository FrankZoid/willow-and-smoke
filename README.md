# Willow & Smoke Kitchen Sample Website Assets

This folder contains the simplified sample asset set for a low-cost restaurant website build. It is intended to reflect the type of content and files a small restaurant client may realistically provide.

## Purpose

Use these files to test a simple restaurant website build with a standard content structure. The assets are for a fictitious restaurant, **Willow & Smoke Kitchen**, and can be replaced with real client content during production.

## Folder Structure

```text
copy/
  about-parking-specials.md
  website-copy.md

data/
  business-info.json
  online-presence-links.json
  ordering-reservations.json

images/
  willow-smoke-exterior-hero-placeholder.jpg
  willow-smoke-brisket-entree-placeholder.jpg
  willow-smoke-salmon-entree-placeholder.jpg
  willow-smoke-dining-room-placeholder.jpg
  willow-smoke-craft-cocktail-placeholder.jpg

logo/
  willow-and-smoke-logo.jpg

menu/
  willow-smoke-sample-menu.pdf
```

## Included Assets

### Business Information

File: `data/business-info.json`

Includes the restaurant name, one-line description, address, phone number, email address, hours of operation, and holiday hours note.

### Website Copy

File: `copy/website-copy.md`

Includes website-ready copy for the hero section, visit/contact section, hours, menu note, online presence links, reservation and ordering links, and About section.

### About, Parking, and Specials Copy

File: `copy/about-parking-specials.md`

Includes optional supporting copy for the About section, parking details, and sample specials.

### Menu PDF

File: `menu/willow-smoke-sample-menu.pdf`

This represents the client-supplied PDF menu. For this low-cost website model, the assumption is that the client provides a PDF menu rather than editable menu data.

### Logo

File: `logo/willow-and-smoke-logo.jpg`

This represents a simple logo file supplied by the client. The assumption is that many small-business clients may have only a basic JPG or PNG logo, not a full professional logo package.

### Photos

Folder: `images/`

Includes five placeholder image assets for layout testing:

1. Exterior / hero image
2. Brisket entrée image
3. Salmon entrée image
4. Dining room / ambiance image
5. Craft cocktail image

These images are placeholders for testing the website build. In a real project, they would be replaced with client-provided photos or AI-generated sample photos when appropriate.

### Online Presence Links

File: `data/online-presence-links.json`

Includes only the online profiles assumed for this simplified sample set:

- Google Maps listing URL
- Google Place ID
- Yelp profile URL

No TripAdvisor, Instagram, or Facebook assets are included in this simplified version.

### Ordering and Reservations

File: `data/ordering-reservations.json`

Includes sample third-party links for:

- Resy reservations
- Toast online ordering

These represent the type of third-party links a restaurant client may provide.

## Build Assumptions

This simplified package assumes a practical, low-cost restaurant website build where the client may provide only the basics:

- Business name and contact information
- Hours
- PDF menu
- A simple logo file
- A few photos
- Google Maps and Yelp links
- Reservation and online ordering links, if available
- Short About, parking, and specials copy, if available

The package intentionally does not include a full brand system, multiple logo formats, CSS files, formal design guidelines, or social media assets.

## Suggested Website Sections

A simple one-page restaurant website could use these assets in the following order:

1. Hero section
2. About / one-line introduction
3. Menu section with PDF link or embedded PDF
4. Featured food or gallery section
5. Reservations and online ordering calls to action
6. Hours and location
7. Parking notes
8. Specials
9. Google Maps / Yelp links

## Replacement Notes for Real Client Projects

When using this structure for a real client, replace the sample files with client-provided assets while keeping the same general folder structure and filenames when possible. This makes the website build easier to repeat across similar projects.

If the client does not provide certain optional items, the site can still be launched with the essential assets: business information, hours, menu PDF, logo, photos, and contact/location details.
